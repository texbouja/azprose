## 3. Socle backend Typst — 🟢 solide

`src-tauri/src/typst_engine/mod.rs`, sous feature flag `typst` (activée par défaut). Compile
proprement (`cargo check` avec et sans la feature). Aucun `TODO`/`unimplemented!()`.

**`RenderWorld`** ([`mod.rs:38`](../src-tauri/src/typst_engine/mod.rs#L38)) — base robuste pour
un éditeur de documents (pas juste un rendu mono-source) :
- résout le `project_root` (accès aux fichiers/images relatifs au document) ;
- charge les packages depuis le cache XDG (`~/.cache/typst/packages`,
  `~/.local/share/typst/packages`) ;
- fonts embarquées + système via `typst-kit`.

**Commandes disponibles :**

| Commande | Signature | Usage frontend |
|---|---|---|
| `typst_render` ([:156](../src-tauri/src/typst_engine/mod.rs#L156)) | `(file_path, source) → SVG fusionné` (`svg_merged`) | ⚠️ **présent mais inutilisé** — c'est la brique de la future preview SVG |
| `typst_export_pdf` ([:163](../src-tauri/src/typst_engine/mod.rs#L163)) | `(file_path, source, path) → ()` | ✅ câblé (toggle preview + export) |
| `typst_page_count` ([:181](../src-tauri/src/typst_engine/mod.rs#L181)) | `(file_path, source) → usize` | inutilisé |

**Points d'attention pour la suite :**
- 🪟 ✅ **Condition essentielle Windows — corrigé** : la résolution des packages
  ([`mod.rs`](../src-tauri/src/typst_engine/mod.rs)) lisait `XDG_*`/`HOME` (→ `None` sous
  Windows) ; elle utilise désormais le crate `dirs` (`%APPDATA%`/`%LOCALAPPDATA%`). Rappel :
  le crate `typst` **compile** sous Windows ; le risque résiduel est *runtime*. Le **public
  cible est majoritairement sous Windows** — condition essentielle dans
  [`editeur_typst.md`](editeur_typst.md).
- Les **warnings de compilation sont jetés** (`compile()`, [:152](../src-tauri/src/typst_engine/mod.rs#L152))
  → à exposer (une commande `typst_preview` renvoyant SVG + warnings en une compilation).
- **Pas de téléchargement de packages** : `#import "@preview/..."` n'aboutit que si déjà en
  cache local. Limitation à documenter / chantier futur.
- Pas de cache Rust ; `typst_check_page_break` (pagination auto) non implémenté — hors MVP.

## 4. Éditeur Typst — 🟡 état du MVP (working tree)

Le mode éditeur `.typ` est déjà bien avancé :

| Élément | Emplacement | État |
|---|---|---|
| Coloration syntaxique `.typ` (CodeMirror `StreamParser`) | [`editor-languages.ts`](../src/lib/editor-languages.ts) (`typst`, l.36/131) | ✅ |
| Boutons preview/export (visibles si `.typ`) | [`Breadcrumb.svelte:264`](../src/components/chrome/Breadcrumb.svelte#L264) | ✅ |
| i18n `breadcrumb.typstPreview` / `typstExport` | `src/locales/*.json` | ✅ (4 locales) |
| Toggle preview + recompilation debouncée | [`app.svelte:1161`](../src/app.svelte#L1161), effect [:1199](../src/app.svelte#L1199) | ⚠️ via PDF |
| Export PDF (à côté du source) | `handleTypstExportPdf` [:1182](../src/app.svelte#L1182) | ✅ fonctionnel |

**Limite structurante — la preview passe par le PDF.** `handleToggleTypstPreview` compile tout
le `.typ` en **PDF** dans `appDataDir`, puis l'affiche via `LazyPdfViewer`
([`app.svelte:1371`](../src/app.svelte#L1371)), avec recompilation debouncée (800 ms) à chaque
frappe. C'est lourd (recompile + re-render PDF.js, perte du scroll) et n'exploite pas le
`typst_render → SVG` déjà disponible.

→ **Recommandation (Phase 1 de `editeur_typst.md`)** : nouvelle commande `typst_preview`
(SVG + warnings, une compilation), composant `TypstPreview.svelte` en layout split, retrait du
chemin PDF-pour-preview. L'export PDF (`typst_export_pdf`) reste inchangé.
